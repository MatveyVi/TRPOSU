# Курсовая работа: Система проката транспорта

## Общая постановка

Информационная система должна обеспечивать возможность брать на прокат, возвращать и регистрировать транспорт.

## 13.1

- Есть 2 точки проката: А, Б.
- Есть 3 типа транспортных средств: а, б, в.
- На каждой точке есть определенное количество каждого транспорта, допустим по 2.
- Есть ограничение на суммарное количество транспорта в точке, например 8.
- На экране должны быть:
  - два раскрывающихся списка;
  - три взаимосвязанных чек бокса (может быть выбран только один);
  - поле текстового вывода;
  - две кнопки: «снять», «вернуть».
- Первый раскрывающийся список выбирает точку проката.
- Чек боксы выбирают тип транспорта.
- По кнопке «снять», если на точке есть транспорт нужного типа, транспорт считается снятым и ему присваивается уникальный идентификатор.
- При ошибке ввода сообщение ошибки выводится в поле текстового вывода.
- Второй раскрывающийся список содержит идентификаторы всех снятых транспортных средств.
- По кнопке «вернуть» транспортное средство должно быть возвращено в выбранную точку проката.
- Можно вернуть транспортное средство, снятое в другой точке.
- Если на точке нет свободных мест, должна быть показана соответствующая ошибка.

### Дополнительное задание к 13.1

- Реализовать запись в файл информации о наличии транспортных средств в точках проката.
- Записывать идентификаторы снятых транспортных средств.
- Подгружать информацию при старте программы.

## 13.2

- Чек боксы заменить на радиокнопки.
- Добавить текстовое поле ввода количества транспортных средств, которые человек хочет снять.
- Проверять правильность ввода и наличие соответствующего количества транспорта нужного вида.
- Добавить progress bar для каждой точки проката.
- Progress bar должен отображать наполненность точек.
- Добавить чек бокс «купить».
- Если «купить» активен, кнопка «снять» вычитает выбранное количество транспорта, не выдавая им идентификатора и без необходимости возврата.

### Дополнительное задание к 13.2

- Расширить запись в файл в соответствии с условием задачи.

## 13.3

- Все ошибки, обрабатываемые ранее, должны быть переведены на всплывающие окна.
- Добавить на основной экран три кнопки:
  - «добавить транспортное средство»;
  - «снять транспортное средство»;
  - «состояние пункта».
- По кнопке «добавить транспортное средство» должно открываться диалоговое окно:
  - радиокнопки с типами транспортных средств;
  - spin box для ввода количества ТС;
  - раскрывающийся список выбора точки проката;
  - кнопки «подтвердить», «отменить».
- Кнопка «отменить» закрывает окно.
- При правильном вводе:
  - число ТС больше нуля;
  - на точке проката достаточно места;
  - кнопка «подтвердить» добавляет соответствующее количество ТС на выбранную точку.
- При ошибке ввода вызывается всплывающее окно с текстом ошибки и кнопкой «ok».
- По кнопке «снять транспортное средство» в главном окне открывается диалоговое окно:
  - аналогичное прошлому окну;
  - дополнительно есть чек бокс «купить».
- Работа окна схожа с описанной выше, разница только в выполняемой функции: прокат или продажа ТС.
- Убрать возможность снимать или покупать транспортное средство в главном окне.
- По кнопке «состояние пункта» открывается диалоговое окно:
  - раскрывающийся список выбора пункта;
  - текстовое поле вывода;
  - progress bar, отображающий общую наполненность точки;
  - кнопка «назад».
